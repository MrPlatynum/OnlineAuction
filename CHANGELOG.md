# История изменений

Все значимые изменения проекта «Лотус» документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [Unreleased]

## [1.0.1] - 2026-06-17

### Изменено
- Файлы выпускной квалификационной работы в `docs/` заменены на финальную сборку: реальный титульный лист, автособираемое оглавление, сквозная нумерация страниц.

## [1.0.0] - 2026-06-17

### Добавлено
- Очередь исходящих писем (outbox) с устойчивым хранением, экспоненциальной задержкой повторов и идемпотентной доставкой уведомлений.
- Пагинация ленты уведомлений (offset-конверт) и единая утилита расчёта `total_pages` для всех постраничных эндпоинтов.
- Ограничение частоты (rate limiting) для счётчика непрочитанных, эндпоинтов отзывов и сброса пароля.
- Микрозамер веерной рассылки уведомлений и покрытие WebSocket-роутера и жизненного цикла outbox-воркера тестами.
- Проверка зависимостей на известные уязвимости через `pip-audit` в CI.
- README проекта с описанием архитектуры, стека, инструкциями по запуску и тестированию.
- Текст выпускной квалификационной работы (DOCX и PDF) в каталоге `docs/`.

### Изменено
- Все колонки с датой и временем переведены в timezone-aware.
- Имя пользователя приводится к нижнему регистру на каждой входной границе.
- Веерная рассылка уведомлений и обновления профиля свёрнуты в меньшее число запросов; диспетчеризация каналов выполняется конкурентно.
- Строковые колонки ограничены разумной длиной; денежные суммы отклоняются на уровне Pydantic при значении мельче копейки.
- Мутация баланса и строка аудита объединены в одну операцию `apply_balance_delta`.

### Исправлено
- Гонка при размещении ставки: лот замораживается на время обработки, сортировка ставок стабилизирована (тай-брейк по `Bid.id`).
- Защита завершения торгов и антиснайпинга от вырожденных входных данных; цепочка продлений ограничена.
- Изоляция блокировок строк outbox в отдельных сессиях; откат планировщика защищён от отмены в момент await.

### Безопасность
- Срок жизни JWT сокращён, зафиксированы клеймы `iss` и `aud`, ужесточён контроль версии токена.
- Удалён запасной путь передачи токена через query-строку на `/ws/notifications`.
- Порог времени ответа при сбросе пароля поднят до 500 мс (защита от тайминг-атак).
- Dev-экземпляр Postgres привязан только к `127.0.0.1`; ужесточена гигиена аутентификации и загрузки файлов.
- Обновлены зависимости для устранения уязвимостей: `python-multipart` до 0.0.31, `PyJWT` до 2.13.0.

## [0.3.0] - 2026-04-29

### Добавлено
- Автозагрузка переменных окружения из `.env` через `python-dotenv` при импорте конфигурации.
- Рабочий процесс CI на GitHub Actions.
- `pytest.ini` с настройкой `pythonpath`, чтобы `pytest` находил пакет приложения без `python -m`.

### Изменено
- Слой работы с базой данных переведён на асинхронный (`asyncpg` + `AsyncSession`).
- Отправка писем сделана неблокирующей.
- Зафиксирована версия `greenlet` (зависимость асинхронного SQLAlchemy).

## [0.2.0] - 2026-04-29

### Добавлено
- Набор тестов на `pytest`, покрывающий аутентификацию, аукционы, ставки и операции с балансом.
- Ограничения целостности на уровне базы данных.

### Изменено
- Миграция хранилища с SQLite на PostgreSQL с управлением схемой через Alembic.
- Обязательная переменная окружения `AUCTION_SECRET_KEY`; вывод через `print` заменён на логирование.

### Исправлено
- Гонка при конкурентных ставках и перерасход баланса.
- Точность хранения денежных сумм.
- Использование устаревших API работы с датой и временем.

## [0.1.0] - 2026-04-29

### Добавлено
- Первая версия платформы электронных торгов на FastAPI: каталог лотов, ставки, аккаунты, базовый цикл аукциона.
